# 📷 آتلیه معین — سیستم مدیریت جامع

سیستم مدیریت حرفه‌ای آتلیه عکاسی و فیلمبرداری عروسی، ساخته‌شده با **Next.js 15** و **TypeScript**.

---

## ✨ امکانات

### مدیریت مراسمات
- رزرو سریع مراسم از طریق تقویم جلالی یا فرم مستقل
- پشتیبانی از چند مراسم در یک روز
- قرارداد کامل با جزئیات: نوع، تاریخ، ساعت، آدرس، مبلغ
- نمایش مبالغ با فرمت فارسی (جداکننده هزارتایی) و تبدیل عدد به حروف
- مدیریت وضعیت: رزرو / جاری / تکمیل / لغو

### مدیریت کارمندان
- پروفایل کامل با ساعت کاری، ساعت حضور، توضیحات شغلی
- لیست وظایف روزانه با نوار پیشرفت رنگی
- تخصیص کارمندان به هر مراسم با شرح نقش

### تقویم جلالی هوشمند
- نشان‌دهی رنگی روزها: آبی = ۱ مراسم، بنفش = ۲+ مراسم
- نشان مناسبت‌های رسمی ایرانی (نقطه قرمز + tooltip)
- رزرو سریع با کلیک روی روزهای خالی (مدیر)

### احراز هویت و نقش‌ها
- ورود امن با JWT ذخیره‌شده در HttpOnly Cookie
- نقش‌ها: `admin` / `accountant` / `employee`
- ریدایرکت خودکار بر اساس نقش

### تجربه کاربری
- **Dark Mode** بدون flicker با `next-themes`
- طراحی RTL کاملاً ریسپانسیو (موبایل / تبلت / دسکتاپ)
- اعلان‌های toast با Sonner (پشتیبانی از dark mode)
- PWA Ready با manifest و آیکون‌های مناسب

## 🚀 راه‌اندازی محلی

### پیش‌نیازها
- Node.js >= 18
- npm

### اجرای سریع (فقط دابل‌کلیک)

```
start.bat
```

همین! همه چیز خودکار انجام می‌شود (نصب پکیج‌ها، ساخت دیتابیس، seed، اجرای سرور).

### مراحل دستی (اختیاری)

```bash
npm install
npx prisma generate
npx prisma db push
npx ts-node prisma/seed.ts
npm run dev
```

پس از اجرا، برنامه در آدرس [http://localhost:3000](http://localhost:3000) در دسترس است.

## 🔐 متغیرهای محیطی

فایل `.env` را در ریشه پروژه بسازید:

```env
# کلید مخفی برای JWT (حتماً تغییر دهید)
JWT_SECRET=your-very-secret-key-change-this

# مسیر پایگاه داده SQLite (پیش‌فرض)
DATABASE_URL="file:./dev.db"
```

> **هشدار:** هرگز مقدار `JWT_SECRET` را در مخزن عمومی قرار ندهید.

## � راهنمای فایل‌های اجرایی (BAT)

این پروژه سه فایل BAT دارد که همه کارها را با دابل‌کلیک انجام می‌دهند:

---

### 🟢 `start.bat` — سرور توسعه (Development)

**کاربرد:** اجرای سریع برای توسعه و تست  
**دابل‌کلیک کن → همه چیز خودکار انجام می‌شود**

**کارهایی که انجام می‌دهد:**
1. اگر سرور قبلی روی پورت 3000 مانده → خودکار می‌کُشد
2. وابستگی‌ها را نصب می‌کند (فقط بار اول)
3. Prisma Client را generate می‌کند
4. دیتابیس SQLite را می‌سازد (اگر نباشد)
5. داده‌های اولیه (seed) را بارگذاری می‌کند
6. سرور توسعه را اجرا می‌کند

**خروجی:** سایت روی `http://localhost:3000`  
**برای بستن:** پنجره CMD را ببند یا `Ctrl+C` بزن

---

### 🔵 `Start Production.bat` — سرور پروداکشن (Production)

**کاربرد:** اجرای سایت در حالت نهایی (سریع‌تر، بهینه‌شده)  
**نیاز:** اول باید `start.bat` حداقل یک‌بار اجرا شده باشد (برای ساخت دیتابیس)

**کارهایی که انجام می‌دهد:**
1. اگر سرور قبلی روی پورت 3000 مانده → خودکار می‌کُشد
2. اگر بیلد وجود نداشت → خودکار `npm run build` می‌زند
3. فایل‌های استاتیک و public را کپی می‌کند
4. دیتابیس را به پوشه standalone کپی می‌کند (با بررسی)
5. IP شبکه را تشخیص می‌دهد
6. سرور پروداکشن را اجرا می‌کند

**خروجی:** سایت روی `http://localhost:3000` + آدرس شبکه  
**برای بستن:** پنجره CMD را ببند یا `Ctrl+C` بزن → سرور بلافاصله متوقف می‌شود

> **نکته مهم:** اگر تغییری در کد دادید، پوشه `.next` را پاک کنید تا بیلد جدید بگیرد.

---

### 🟣 `Build Deploy.bat` — ساخت پکیج Deploy

**کاربرد:** ساخت یک پوشه مستقل برای نصب روی سیستم دیگر  

**کارهایی که انجام می‌دهد:**
1. بیلد پروداکشن می‌گیرد
2. standalone + static + public + دیتابیس را در پوشه `deploy/` کپی می‌کند
3. یک `Start.bat` داخل deploy می‌سازد

**نحوه استفاده:**
1. دابل‌کلیک `Build Deploy.bat`
2. پوشه `deploy/` ساخته می‌شود (~80 MB)
3. این پوشه را به هر ویندوزی که **Node.js** دارد کپی کن
4. دابل‌کلیک `Start.bat` داخل پوشه deploy

> **جایگزین:** `node scripts/build-deploy.js` همین کار را انجام می‌دهد.

## 🔧 تغییرات و رفع باگ — نسخه ۱۴۰۵/۰۱/۳۰

### رفع باگ فایل‌های اجرایی (bat)
- **`Start Production.bat`**: مسیر کپی دیتابیس از `prisma\dev.db` (اشتباه) به `prisma\prisma\dev.db` (صحیح) اصلاح شد
- **`Start Production.bat`**: IP شبکه هاردکد شده (`192.168.20.232`) به تشخیص خودکار تغییر یافت
- **`Start Production.bat`**: خروجی سرور هم‌زمان در کنسول و فایل لاگ نمایش داده می‌شود (قبلاً فقط در فایل لاگ بود)
- **`Build Deploy.bat`**: متغیر `DATABASE_URL` به `Start.bat` تولیدشده اضافه شد (بدون آن سرور خطا می‌داد)
- **`Build Deploy.bat`**: مسیر کپی دیتابیس به `deploy\prisma\dev.db` اصلاح شد
- **`scripts/build-deploy.js`**: انکودینگ `Start.bat` از `ascii` به `utf-8` تغییر یافت (کاراکترهای فارسی خراب نشوند)
- **`scripts/build-deploy.js`**: `DATABASE_URL` و `COOKIE_SECURE` به اسکریپت استارت اضافه شد

### رفع مشکلات امنیتی
- **سطح دسترسی API**: بررسی نقش ادمین به تمام عملیات نوشتنی (ایجاد/ویرایش/حذف) اضافه شد:
  - مراسمات: `POST`، `PUT`، `DELETE`
  - پرداخت‌ها: `POST`
  - تخصیص کارمندان: `POST`، `DELETE`
  - پلن‌ها: `POST`، `PUT`، `DELETE`
  - تنظیمات: `PUT`
  - ویرایش کارمندان: `PUT`
- **`isAdmin`**: نقش `accountant` (حسابدار) نیز دسترسی مدیریتی دارد (هماهنگ با middleware)
- **`register` API**: رمز عبور هش‌شده دیگر در پاسخ ارسال نمی‌شود
- **`logout` API**: فلگ `secure` کوکی با `COOKIE_SECURE` محیطی هماهنگ شد (قبلاً `NODE_ENV` چک می‌شد)
- **حذف کارمند**: عملیات در یک `$transaction` انجام می‌شود (جلوگیری از داده‌های یتیم)

### رفع باگ‌های عملکردی
- **مراسمات کارمند**: رفع باگ تکرار مراسمات هنگامی که کارمند چند وظیفه در یک مراسم دارد
- **ورود ارقام فارسی**: در فرم مراسمات، ارقام فارسی (۰-۹) و عربی (٠-٩) به‌درستی تبدیل به عدد لاتین می‌شوند
- **ساعت حضور**: رفع `parseFloat("14:30") = 14` — فرمت `HH:MM` به عدد اعشاری تبدیل می‌شود (مثلاً `14:30` → `14.5`)
- **SWC Build**: پکیج `@next/swc-win32-x64-msvc` نسخه ناسازگار (16.x) حذف شد — بیلد به‌درستی انجام می‌شود

### دیباگ روی گوشی واقعی

از موبایل از طریق IP شبکه وصل شوید (هنگام اجرا نمایش داده می‌شود):

```
http://192.168.x.x:3000
```

لاگ ورودها در `logs/access.log`:

```
[2026-04-19T10:23:11Z] LOGIN OK   user="admin" role=admin IP:192.168.20.15
[2026-04-19T10:23:15Z] LOGIN FAIL user="admin" IP:192.168.20.15
```

اگر از موبایل لاگین نمی‌شود:
1. `logs/access.log` را بررسی کنید
2. `LOGIN FAIL` → رمز اشتباه
3. هیچ entry → موبایل به سرور دسترسی ندارد (IP چک شود)
4. `LOGIN OK` ولی ریدایرکت نمی‌شود → مطمئن شوید `COOKIE_SECURE=false` در `.env` است
